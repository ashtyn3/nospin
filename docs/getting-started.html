Qoute

Basics of Qoute

Thank you for reading the docs! It takes so long to write this so thank you for taking the time to read it. Qoute is a simple filesystem designed to make the cloud more accessible. Qoute allows you to synchronize all of your devices file systems into one. Here is an example:

Lets say Marvin has a computer for work and a computer for home as a lot people do. Well one day Marvin forgot his work computer at work (Ooh No). With Qoute All Marvin has to do is open his home computers terminal and grab all files he needs from Qoute to continue his work from home.

Qoute usage

So far Qoute has your basic options for any filesystem, read, write, and delete. To get started using Qoute first install the latest version from the github releases. Once you downloaded the zip file that should have a name that looks similar to Qoute-1.1.0.zip. Extract that file and install the binary file to your computer currently Qoute only supports Mac and Unix based operating systems.

To make sure Qoute is working on your computer run:

Qoute

It should return something that looks like this:

Usage:
Qoute COMMAND
commands:
    -auth string                   Generates user with passed in email.
    -put, -p string [-o string]    Put file with path of -o option or with passed in path.
    -get, -g string                Get file with path.
    -del, -D string                Delete file with path.
    -user, -u                      Prints information about current user.

Before you start uploading stuff you will need to register a user. You can do this with the following: 

Qoute -auth marvin@example.com

General rule of thumb is when making a Qoute user always use your primary email address as the username. The auth command will return your public key and your email.

To test to make sure the user was created successfully you can run: 

Qoute -user

This should return your public key and email.

File methods

Qoute works off tries to duplicate your original file structure across all the cloud. So when uploading a file pass in a relative file path and Qoute will use that path as the path to the file.

For example:

Lets say you have the following directory structure:

example_dir
├── Documents
│  └── Essay.txt
└─── Pictures
   └── family_christmas_2019.jpeg

Lets say you want to upload Essay.txt to Qoute and your currently in the example_dir directory. Well all you have to do is: 

Qoute -put marvin@example.com/Documents/Essay.txt

This will upload the file to Qoute with the name Documents/Essay.txt. If you wanted to do a custom file name for when its uploaded just include a -o flag. For example you can do: 

Qoute -put marvin@example.com/Documents/Essay.txt -o essay.md

This will upload the file Documents/Essay.txt as essay.md.

To get a file you can just do that same thing as put except with -get. 

Qoute -get marvin@example.com/essay.md

The get method will print out the contents of that file to the console. If you don't want this to happen and instead you want to write the contents to a file just do the following: 

Qoute -get marvin@example.com/essay.md > newFile.md

This will write the contents of the essay.md in Qoute to the file newFile.md on your local computer.

You can also get a folder with the following.

Qoute -get marvin@example.com/someFolder/

This will return the names of the files in the folder someFolder on Qoute.

To delete a file is really simple all you have to know is the name of the file you want to delete. 

Qoute -del marvin@example.com/essay.md

This will delete the file at the path specified on Qoute.
